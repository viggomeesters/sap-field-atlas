# Code of Conduct

SAP Field Atlas uses a compact contributor standard: be precise, constructive, and public-safe.

## Expected behavior

- Be respectful and professional.
- Challenge data quality, sources, and assumptions directly, without personal attacks.
- Keep discussions focused on generic SAP object knowledge, repository quality, and validation behavior.
- Respect the public-data boundary: do not pressure contributors to disclose customer/project-specific context.

## Unacceptable behavior

- Harassment, insults, threats, or discriminatory language.
- Posting customer data, credentials, internal URLs, screenshots, exports, or proprietary mappings.
- Copying proprietary SAP documentation into the repository.
- Repeatedly ignoring maintainer requests about scope, safety, or source quality.

## Enforcement

Maintainers may edit, hide, close, or delete contributions that violate this policy, and may block contributors who repeatedly violate it.

For sensitive reports, do not paste private data into a public issue. Use the private contact route described in `SECURITY.md`.
