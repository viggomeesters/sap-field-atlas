# Security Policy

SAP Field Atlas is public and must contain only generic, non-customer SAP knowledge.

## Reporting

Open a GitHub issue for ordinary public-data-boundary problems. If a report contains sensitive data, do not paste the data into an issue; contact the repository owner privately through GitHub instead.

## Sensitive data boundary

Do not contribute:

- customer/client names;
- SAP screenshots, extracts, table exports, logs, or dumps from customer systems;
- internal URLs, ticket ids, project ids, credentials, or proprietary mappings;
- copied proprietary SAP documentation.

If sensitive content is accidentally committed, stop normal work and remove it from the public repo history before adding more commits.
